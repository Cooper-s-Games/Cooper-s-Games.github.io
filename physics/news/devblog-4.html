Dev Blog 4

Gamemodes, UI, and stuff I've been experimenting with
Developer access
I've given developer access out to a few people. It's been really helpful to see a bunch of bugs to fix and features to add. I was previously working on a bunch of features that the game really didn't need, so it's been good to get back on track and work on stuff for both the future players and developers.
Suicide Barrels
I've been working on a Suicide Barrels gamemode to test AI and ragdolls. I'm using most code from my deathmatch base, so it's been pretty simple to code. I'll probably release the code later on, so people can get some more examples of how gamemodes will be made. While I was testing I made some bots that just run around and shoot/explode randomly.


Sandbox
I started working on a sandbox gamemode. I started with implementing some basic tools like physgun, weld, prop spawner. This is a very early version of it, it's going to go through a lot of work because I know this is going to be the main gamemode people play. It's using the deathmatch base. Deathmatch

Deathmatch is still pretty raw. I've been using it as the base shooter gamemode, but I know there's still a bit of work I need to do on it to make it more deathmatch-y and less devtesting-y.

Custom Imports
Models from Source, are of course, different sizes and have different bones. For example, mossman is a giant and I had to scale her down to 0.025 to make it look normal. So I've been expirementing with custom asset import sizes. You'll be able to have a bunch of options to mess around with. You can even leave it as the default import if you want.
