My goals with importing maps, models, and textures

March 5, 2021 by Cooper J.

Importers

You (most likely) will be able to write your own importers. I want to keep the base importers open source, so people can check them out and make their own.

I don't want to create a custom file format, since that would be tougher to import files with. You can import the usual file types: bsp, fbx, obj, and mdl.

This is part of a basic WIP texture importer, as you can tell it's simpler than the maps. The one problem right now is that you have to manually turn the textures (vtf in my case) into png to make it import.

I don't want to use any Source assets in the full release, I just like playing around with the maps, since HL and HL2 have a large amount of maps and models to test with.

The Goal

My goal, which is a stupid one, is to have a good base importer, and allow people to make their own importers and share them, and have a bunch of different importers so people who don't know how to program can just load a script somebody made, import stuff onto their game, then play it with their friends.
